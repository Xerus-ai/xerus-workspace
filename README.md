# Xerus Workspace Template

Scaffold template for Xerus AI workforce workspaces. Cloned once during workspace creation, then dynamic content is generated by the backend scaffold service.

## What's Static (in this repo)

- **Directory structure**: Full workspace layout with all folders
- **Hook scripts**: 11 SDK lifecycle hooks in `.claude/hooks/scripts/`
- **Commands**: Platform slash commands in `.claude/commands/`
- **Xerus Master identity**: SOUL.md, BOOTSTRAP.md, STATUS.md, config.json
- **Memory structure**: `.system/.memory/` with index and base files
- **Shared spaces**: office, standup, knowledge, inbox

## What's Dynamic (generated by scaffold)

- **CLAUDE.md** (root): Generated from `root-claude-md.template.ts` with user-specific content
- **`.claude/settings.json`**: Generated with user's model preferences and permissions
- **`.claude/agents/*.md`**: SDK subagent definitions generated from agent config.json files
- **`.mcp.json`**: Platform MCP server config with user's connected tools
- **`data/company.db`**: Empty SQLite with base schema
- **Channel folders**: Created when user adds domains/channels via UI

## Scaffold Flow

```
1. Daytona creates sandbox
2. Clone this repo into /home/daytona/
3. Delete .git (workspace is not a persistent clone)
4. Generate dynamic files (CLAUDE.md, settings.json, .mcp.json, agents/*.md)
5. Initialize .system/.memory/.git (git-memory repo)
6. Create data/company.db with base schema
7. Sync initial metadata to DB
```
