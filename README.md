# Xerus Workspace Template

Scaffold template for Xerus AI workforce workspaces. Cloned once during workspace creation, then dynamic content is generated by the backend scaffold service.

## Marketplace (Git Submodules)

The `marketplace/` directory contains git submodules referencing the agent and skill repos:

```
marketplace/
  agents/  -> git submodule -> github.com/xerus-ai/xerus-agents
  skills/  -> git submodule -> github.com/xerus-ai/xerus-skills
```

Clone with `--recursive` to pull marketplace content:
```bash
git clone --recursive https://github.com/xerus-ai/xerus-workspace.git
```

## What's Static (in this repo)

- **Directory structure**: Full workspace layout with all folders
- **Hook scripts**: 11 SDK lifecycle hooks in `.claude/hooks/scripts/`
- **Commands**: Platform slash commands in `.claude/commands/`
- **Settings**: `.claude/settings.json` with SDK config
- **Xerus Master identity**: SOUL.md, BOOTSTRAP.md, STATUS.md, config.json in `agents/xerus-master/`
- **Memory structure**: `.memory/` with index and base files
- **Shared spaces**: office, standup, knowledge, inbox
- **Marketplace catalog**: Agent and skill definitions (via submodules)

## What's Dynamic (generated at scaffold time)

- **CLAUDE.md** (root): Generated from `root-claude-md.template.ts` with user-specific content
- **`agents/{slug}/CLAUDE.md`**: Module CLAUDE.md generated from DB (KB, skills, tools, colleagues)
- **`agents/{slug}/config.json`**: Generated from DB agent row
- **`agents/{slug}/SOUL.md`**: Generated from DB personality data
- **`.claude/agents/*.md`**: SDK subagent definitions generated from agent config
- **`.mcp.json`**: Platform MCP server config (runner path is dynamic)
- **`data/company.db`**: Empty SQLite initialized with base schema
- **Channel folders**: Created when user adds domains/channels via UI

## Scaffold Flow

```
1. Daytona creates sandbox
2. git clone --recursive xerus-workspace into /home/daytona/
3. Delete .git and marketplace/*/.git (workspace is not a persistent clone)
4. Generate dynamic files (CLAUDE.md, agent configs, .mcp.json)
5. Initialize .memory/.git (git-memory repo)
6. Create data/company.db with base schema
7. Sync initial metadata to DB
```

## Platform Updates

Platform-owned files (hooks, commands, settings, rules) can be updated without touching user data. See `daytona-first-workspace.md` Platform Updates section.
